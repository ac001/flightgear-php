The $Site object is the main controller class that is accessible in php and tempaltes.

Creating a new site area is a simple case of some simple configuration and scripts

A $Site object needs to be declared and properties set.

{literal}
$Site = new fgSite();
$Site->id = 'foo';
$Site->title = 'Foo Fun';{/literal}

$Site->id

The $Site->id property is used as

  • Is a kinda namespace
  • Compiled smarty templates as saved with this as the Site key
  • The templates will be used from templates/$Site->id/
  • Is matched against the "inter site navigation" at the top

$Site->title

The $Site->title property appears in the header bar and the html title

$Site->section

$Site->page

The current section and page, these are currently set in the top level script eg

//* $page can be null, $section defaults to "index"
$Site->section = isset($_REQUEST['section']) ? $_REQUEST['section'] : 'index';
$Site->page = isset($_REQUEST['page']) ? $_REQUEST['page'] : null;

$Site->addPageNav()

The $Site->addPageNav() function is used to add "sections" and "pages" to a site.

The arguments take the form
$section, $nav_label, $page_title, array($pages)

  • $section - this is a top level of navigation
  • $nav_label - The navigation and $page_title if $page_title is null
  • $page_title - if null, then $nav_label is used as page title
  • array($pages) - sub_pages in the form $page, $nav_label, $optional_title
  • Potentially later a multi-tier system and css might be interesting.

$Site->pageTitle()

This function outputs the current page's title.

$Site->contentTemplate()

Returns the current coentent template
