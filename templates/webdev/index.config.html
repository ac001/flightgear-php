The "application" is designed to be as self configurable as possible, however here are some pointers to how its achieved at the moment.

config/config.inc.php

The config.inc.php is the main config file that needs to be included at the top of each script. It defines the enviroment, set's up the autoload enviroment, declares constants, etc.

The file is intended for use both in a webpage enviroment, and also as a shell, eg cron

Two variables $REQUIRE_SMARTY and $REQUIRE_DB need to be declared and true before the config file is included to load smarty and connect to a database respectively.

{source lang='php'} $REQUIRE_SMARTY = 1; // we need smarty for this web page $REQUIRE_DB = 1; // don't need database require_once('config/config.inc.php');{/source}

SITE_ROOT

This constant is the full system path to the "top" level of the site, ie the root

{source lang='php'}echo SITE_ROOT; // outputs /home/user_name/public_html/flightgear-php/{/source}

config/*.ini

Other configurations such as mpservers, mirrors, irc channels etc are stored in good olde *.ini files. However the structure is such that these could be abandoned for an alternate (eg a database) later should the need require.

Current ini files include

  • mirrors.ini
  • irc.ini
  • mpservers.ini

There is a security consideration to be taken into account with the db configuration.
